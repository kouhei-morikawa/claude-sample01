# React コーディング学習カリキュラム

初心者から実践レベルまで、ステップアップ式で React を学ぶためのサンプル集です。  
各サンプルは独立したフォルダ（`src/samples/`）に格納し、一つひとつ動かして理解を深めます。

---

## 📂 フォルダ構成（予定）

```
src/
└── samples/
    ├── 01_hello_world/          # Lv.1 最初のコンポーネント
    ├── 02_props/                # Lv.1 Props（プロパティ）の基本
    ├── 03_state_counter/        # Lv.2 useState でカウンター
    ├── 04_event_handling/       # Lv.2 イベントハンドリング
    ├── 05_conditional_render/   # Lv.2 条件分岐レンダリング
    ├── 06_list_render/          # Lv.2 リストのレンダリング
    ├── 07_forms/                # Lv.3 フォームと制御コンポーネント
    ├── 08_useeffect/            # Lv.3 useEffect の基本
    ├── 09_fetch_api/            # Lv.3 API 通信（fetch）
    ├── 10_custom_hooks/         # Lv.4 カスタムフック
    ├── 11_context/              # Lv.4 Context API で状態共有
    ├── 12_reducer/              # Lv.4 useReducer で複雑な状態管理
    ├── 13_memo_callback/        # Lv.5 React.memo / useCallback / useMemo
    ├── 14_router/               # Lv.5 React Router でページ遷移
    └── 15_todo_app/             # Lv.5 総合演習：Todo アプリ
```

---

## 📚 ステージ別カリキュラム

### ステージ 1：React の基礎（Lv.1）

| # | サンプル名 | 学習テーマ | ポイント |
|---|-----------|------------|---------|
| 01 | hello_world | 最初のコンポーネント | JSX の書き方、コンポーネントの定義と表示 |
| 02 | props | Props の基本 | 親→子へのデータ渡し、Props の型定義（TypeScript） |

**習得目標：**
- JSX で UI を記述できる
- コンポーネントを関数で定義できる
- Props を使ってコンポーネントを再利用できる

---

### ステージ 2：状態管理の入口（Lv.2）

| # | サンプル名 | 学習テーマ | ポイント |
|---|-----------|------------|---------|
| 03 | state_counter | useState カウンター | state の基本、再レンダリングの仕組み |
| 04 | event_handling | イベントハンドリング | onClick / onChange など各種イベント |
| 05 | conditional_render | 条件分岐レンダリング | `&&` / 三項演算子 / コンポーネントの出し分け |
| 06 | list_render | リストのレンダリング | `map()` でリスト表示、`key` の重要性 |

**習得目標：**
- `useState` で動的な UI を作れる
- ボタンや入力フォームのイベントを処理できる
- 条件やリストに応じた表示ができる

---

### ステージ 3：副作用と非同期処理（Lv.3）

| # | サンプル名 | 学習テーマ | ポイント |
|---|-----------|------------|---------|
| 07 | forms | フォームと制御コンポーネント | input 要素を state で管理する |
| 08 | useeffect | useEffect の基本 | マウント時・更新時・クリーンアップ |
| 09 | fetch_api | API 通信（fetch） | 非同期処理、ローディング状態、エラーハンドリング |

**習得目標：**
- フォームの入力値を state で管理できる
- `useEffect` のライフサイクルを理解できる
- 外部 API からデータを取得して表示できる

---

### ステージ 4：設計・アーキテクチャ（Lv.4）

| # | サンプル名 | 学習テーマ | ポイント |
|---|-----------|------------|---------|
| 10 | custom_hooks | カスタムフック | ロジックの分離・再利用 |
| 11 | context | Context API | グローバルな状態共有、Provider / Consumer パターン |
| 12 | reducer | useReducer | 複雑な状態管理、action / reducer パターン |

**習得目標：**
- ビジネスロジックをカスタムフックに切り出せる
- Context で props のバケツリレーを解消できる
- `useReducer` で予測可能な状態遷移を実装できる

---

### ステージ 5：パフォーマンスと応用（Lv.5）

| # | サンプル名 | 学習テーマ | ポイント |
|---|-----------|------------|---------|
| 13 | memo_callback | 最適化 | `React.memo` / `useCallback` / `useMemo` の使い所 |
| 14 | router | React Router | SPA のルーティング、動的パス、ネストルート |
| 15 | todo_app | 総合演習 | これまでの学習を組み合わせた実践アプリ |

**習得目標：**
- 不必要な再レンダリングを防ぐ最適化ができる
- SPA でページ遷移を実装できる
- 複数の概念を組み合わせてアプリを設計・実装できる

## 🛠️ 各サンプルの構成（テンプレート）

各フォルダは以下のファイルで構成します：

```
src/samples/XX_sample_name/
├── index.tsx        # サンプルのエントリーポイント（App から読み込む）
├── README.md        # このサンプルの解説（日本語）
└── （必要に応じて追加コンポーネントや hooks）
```

---

## 🚀 進め方

1. `LEARNING.md`（本ファイル）でカリキュラム全体を確認する
2. 実装したいサンプルのフォルダを `src/samples/` 以下に作成する
3. 各フォルダの `README.md` を先に読んでからコードを書く
4. `npm run dev` で動作確認しながら学習する

---

## 📝 備考

- コードとコメントはすべて **日本語** で記述します
- TypeScript を使用します（型定義も学習範囲）
- スタイリングは既存の CSS を流用、または最小限の inline style を使用します
- 外部ライブラリは原則として追加せず、React 本体の機能に集中します（Router のみ例外）
