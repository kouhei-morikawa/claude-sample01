# サンプル05：条件分岐レンダリング

## 何を学ぶか

- **&&** 演算子による条件付き表示
- **三項演算子**（`? :`）による出し分け
- **変数**への JSX 代入を使った条件分岐
- コンポーネント自体を条件によって変える方法

---

## コードの解説

### ① 三項演算子（`? :`）

「AかBか」どちらかを表示したいときに使います。

```tsx
{isLoggedIn
  ? <p>ようこそ！</p>    // true のとき
  : <p>ログインしてください</p>  // false のとき
}
```

### ② && 演算子

「条件を満たすときだけ表示する」ときに使います。

```tsx
{isLoggedIn && <UserMenu />}
```

⚠️ **落とし穴**：左辺に `0` や `NaN` などの falsy 値を使うと、その値が画面に表示されます。

```tsx
// ❌ count が 0 のとき、"0" が画面に表示される
{count && <p>アイテムが {count} 件あります</p>}

// ✅ !! で真偽値に変換する
{!!count && <p>アイテムが {count} 件あります</p>}

// ✅ または比較演算子を使う
{count > 0 && <p>アイテムが {count} 件あります</p>}
```

### ③ 変数への JSX 代入

複雑な条件は JSX の外で変数に代入しておくと読みやすくなります。

```tsx
// JSX の外で条件を処理
let message: React.ReactNode
if (status === 'loading') {
  message = <p>読み込み中...</p>
} else if (status === 'error') {
  message = <p>エラーです</p>
} else {
  message = <p>成功！</p>
}

// JSX 内では変数を使うだけ
return <div>{message}</div>
```

### ④ null を返すとコンポーネントを非表示にできる

```tsx
function ConditionalComponent({ show }: { show: boolean }) {
  if (!show) return null  // 何も表示しない
  return <p>表示されています</p>
}
```
